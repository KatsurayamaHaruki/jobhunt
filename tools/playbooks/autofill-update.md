# プレイブック: 新しい設問が出たとき（台帳更新 → オートフィル再生成）

フォーム自動入力は**完全機械処理**（`tools/build-autofill.mjs` が生成したスクリプト）。
ランニングのトークンはゼロ。Claude を使うのは「台帳に無い欄が出たとき」だけ。

## 日常の流れ（人間・トークン0）
1. 各社マイページに自分でログインしフォームを開く。
2. 画面右下の「↧ 台帳入力」ボタン（userscript）か、登録済みブックマークレットを押す。
3. 既知の基本欄が下書きされる。`〔　〕`（台帳が空）の項目は飛ばす＝推測しない。
4. 内容を確認して**送信は自分で**押す。

## 新しい設問・欄が出たとき（ここだけ Claude Code）
1. その欄のラベルと、入れるべき値を Claude に伝える（不明なら一緒に決める）。
2. Claude が `profile-daicho.md` の該当セクションに追記する。
   - 確定値はそのまま、要確認の下書きは `〔値〕` か `※要確認` を付ける（台帳ルール準拠）。
   - パスワード・口座・マイナンバーは**絶対に台帳へ入れない**。
3. フォーム欄のラベルが既存シノニムに無ければ、`tools/build-autofill.mjs` の `SPECS` に
   その台帳ラベルとシノニム（フォームに出る語の部分一致）を1行追加する。
4. 再生成して台帳も同期:
   ```
   node tools/build-autofill.mjs   # スクリプトを作り直す
   node tools/sync-profile.mjs     # 台帳をポータルにも反映
   ```
5. userscript を使っている場合は、Tampermonkey に新しい `public/jobhunt-autofill.user.js`
   を入れ直す（ブックマークレット派は `...-bookmarklet.txt` を貼り直す）。

## 確認用
`public/jobhunt-autofill-dict.json` に、今どの欄をどの値で埋めるかが全部出ている。
意図しない値が入っていないかはここで点検できる。

# 指示書: ブラウザでES作成（Claudeアプリ＋Chrome拡張セッション用）

このファイルは、**ログイン済みブラウザを読めるClaude（Claudeアプリ＋Chrome拡張）**が、
迷わずES作成を完遂するための指示書。端末も jobctl も使わず、**ブラウザとポータルUIだけ**で完結する。

## 登場するもの
- **就活ポータル**: https://job-hunt-pi-orpin.vercel.app/ （Googleログイン済み。企業・締切・ES・自己分析資料が入っている）
- **対象企業のESページ**: ユーザーが別タブで開いている（ログイン済みマイページ）

## あなたの役割
ESページの設問を読み取り、**ポータルの自己分析マスター資料・基本プロフィールを根拠に**下書きを作り、
ユーザー確認後に**ポータルへ保存し、選考ステータスを更新**する。人間の操作は「確認」だけにする。

## 手順（この順で機械的に）

1. **設問を読む**：開いている対象企業のESタブを読み、**設問文・文字数上限・コース/インターン名**を漏れなく一覧化する。
   上限の記載が無ければユーザーに確認（推測しない）。

2. **素材を集める**（ポータルを参照）：
   - ポータルの **「資料」タブ** → 「自己分析マスター資料」と「基本プロフィール」を読む（これが事実の正）。
   - ポータルの **「ES作成」→「設問別（横断）」** で、似た設問の**過去ES**を探す。あれば流用ベースにする（=作り直さない＝節約）。

3. **下書きを書く**（ルール厳守）：
   - 提供資料に**書かれた事実のみ**。無い事実は創作せず `[要確認]` と明示。
   - 結論先行（PREP / STAR）。文体は **だ・である**。
   - 文字数は**上限ギリギリ**を目標、難しくても**最低90%超**。上限超過は禁止。字数は**コードポイント数**で数える（絵文字や全角も1字）。
   - 空疎な美辞麗句・どの企業にも使える志望動機を避け、**具体・定量**。志望動機はそのコース内容に寄せる。
   - 専門用語には短い補足。出力は**本文のみ**。

4. **自己チェックして提示**：各設問の本文と字数を出し、上限内・上限ギリギリ（最低90%超）かを示す。**ユーザーの確認を待つ**。

5. **ポータルへ保存**（確認OK後）：
   - 「ダッシュボード」で対象企業が無ければ、上部の入力欄に社名→Enterで追加。
   - 「ES作成」→「企業別」→ 対象企業を選択 → **「＋ ESを追加」** → 設問・文字数上限・本文を貼って **保存**。設問ごとに繰り返す。

6. **状況を更新**：
   - 対象企業カードの**ステータスのプルダウン**を実態に合わせる（提出したら「ES提出」等）。
   - 締切が分かれば、カードの締切追加（種別プルダウン＋日付＋任意の時刻。時刻なしはその日の23:59扱い）で登録。

## トークン節約の原則
- まず「設問別」で過去ESを探し、近いものは**流用→差分修正**で済ませる（ゼロから生成しない）。
- マスター資料は長いので、設問に関係する節（ガクチカ/研究/志望動機など）を中心に読む。
- 同じ設問を何度も生成しない。一度作ったらポータルに保存し、次回は再利用する。

## 注意
- パスワード・口座・マイナンバーは扱わない／保存しない。
- 生成は素のモデル出力。**提出ボタンは必ず人間が押す**。

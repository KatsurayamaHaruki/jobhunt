# プレイブック: ES下書きの生成（サブスクで実行・API課金なし）

旧版はサーバーから Anthropic API を叩いていた（従量課金）。**廃止した。**
代わりに Claude Code（サブスク）が下書きを書き、`jobctl save-es` でポータルに保存する。

## 前提
- 文脈資料: `profile-daicho.md`（基本台帳）と「自己分析マスター資料」。
  マスター資料はポータルの「資料」タブ、または手元のファイルを Claude Code に読ませる。

## 生成ルール（厳守）
- 提供資料に書かれた**事実のみ**。記載のない事実は創作せず `[要確認]` と明示。
- 結論先行（PREP / STAR）。文体は **だ・である** 調。
- 文字数は**上限の約9割**を目標、上限超過禁止。字数は「コードポイント数（`[...s].length`）」で数える。
- 空疎な美辞麗句・どの企業にも使える志望動機を避け、具体的・定量的に。
- 専門用語には短い補足。

## 手順
1. 設問・文字数上限・対象企業を確認。
2. 上記ルールで下書きを書く。複数案が必要なら案ごとに提示。
3. 字数を自分で数えて上限内（目標9割）に収める。
4. 人間が内容を確認・修正（←ここが唯一の人手）。
5. 確定したら本文をファイルに保存し、ポータルへ:
   ```
   node tools/jobctl.mjs save-es "<企業id|名前>" "<設問>" <上限> --text-file <本文ファイル>
   ```
   保存は `es_drafts` に追記され、`status_log` に証跡が残る。

## メモ
- 生成は素のモデル出力。**必ず人間が確認**してから提出する。
- ポータルの「ES作成」タブは閲覧・微修正用。生成ボタン（課金API）は撤去済み。

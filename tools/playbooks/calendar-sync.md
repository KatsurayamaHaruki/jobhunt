# プレイブック: 締切 → Googleカレンダー同期

**実行者: Claude Code（サブスク）。** ポータルの締切タスクを Google カレンダーにイベント化し、
リマインダーで取りこぼしを防ぐ。Google Calendar 連携ツール（MCP）を使う。

## 手順（締切 → カレンダー）

1. **未同期の締切を取得**
   ```
   node tools/jobctl.mjs tasks
   ```
   → `[{company, label, date, title:"【就活】〇〇｜ES締切", ...}]` が日付順で出る。

2. **重複を避けて作成**
   各タスクについて、Google カレンダーをその日付で検索（`list_events` を date 範囲で）し、
   **同じ `title` のイベントが既にあればスキップ**。無ければ `create_event` で作成する。
   - 件名: タスクの `title`（例 `【就活】第一生命｜Webテスト`）
   - 日付: `date`（終日イベント）。時刻が分かっている場合のみ時間指定にする。
   - リマインダー: 前日 と 当日朝（例 popup 1日前 / 当日9:00）。
   - 説明: 「就活ポータルから自動作成」＋企業名。

3. **報告**：作成/スキップした件数を一覧で示す。

## 手順（カレンダー → ポータル：面接予定の取り込み・任意）
1. `list_events` で就活関連（タイトルに企業名や「面接」を含む）イベントを取得。
2. ポータル未登録の面接/説明会があれば、対応する企業に締切として追加:
   ```
   node tools/jobctl.mjs add-task "<企業名>" "<一次面接 等>" <YYYY-MM-DD> --source manual --evidence "Googleカレンダー: <イベント名>"
   ```

## 重複防止について
イベント件名の規約 `【就活】{企業名}｜{ラベル}` を**一意キー**として使う。
作成前に必ず同名イベントの有無を確認すること（このプレイブックはローカルに同期状態を保存しない）。

## 定期実行
Gmail同期（`gmail-sync.md`）の直後にこれを回すと、新しく増えた締切がそのままカレンダーに乗る。
